Git & GitHub Guide

Personal notes for syncing the vubio.github.io repository.

1. Checking Differences (Who is newer?)

Before doing anything, you need to "fetch" the latest info from GitHub without changing your local files yet.

git fetch origin

To see if you are ahead or behind:

git status
  • "Your branch is up to date": Everything is perfectly synced.
  • "Your branch is ahead of 'origin/main'": Local is newer. You have changes to Push.
  • "Your branch is behind 'origin/main'": Online is newer. You need to Pull.

To see the actual code changes:

git diff main origin/main

2. Make Online similar to Local (Push)

Use this when you have edited files on your laptop and want them to appear on your website.

git add .
git commit -m "Describe your changes here"
git push origin main

Note: If this fails, someone (or you) might have edited the files online directly. Pull first!

3. Make Local similar to Online (Pull)

Use this if you edited files directly on the GitHub website or from another computer.

git pull origin main

This command "grabs" the online version and merges it into your local folder instantly.
