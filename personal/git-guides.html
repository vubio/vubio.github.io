GIT CHEAT CODE

1. Check Status (Who is newer?)

Compare your laptop to the GitHub server.

STEP 1: FETCH LATEST INFO
git fetch origin
STEP 2: CHECK STATUS
git status
  • "Ahead of origin/main": Laptop is NEWER. (Need to Push)
  • "Behind origin/main": GitHub is NEWER. (Need to Pull)
  • "Up to date": Perfect sync.

2. Make Online = Local (Push)

Use this after you finish coding on your laptop.

git add .
git commit -m "Update"
git push origin main

3. Make Local = Online (Pull)

Use this if you edited code on the GitHub website directly.

git pull origin main

4. Clone a Repository (First Time Setup)

Use this to download a GitHub repository to your computer for the first time.

STEP 1: OPEN TERMINAL/COMMAND PROMPT

Navigate to where you want to create the folder. Example: Documents folder

cd ~/Documents
STEP 2: CLONE THE REPOSITORY

Replace username and repo-name with actual values.

git clone https://github.com/username/repo-name.git
STEP 3: ENTER THE FOLDER
cd repo-name

Now you have a local copy and can start working!

5. Fix Credentials Error (Old Account Access)

If you get "Access denied" or "Authentication failed" when pushing/pulling:

PROBLEM

Your computer has git credentials cached from an old GitHub account, blocking access with your current account.

SOLUTION: CLEAR OLD CREDENTIALS

On Mac/Linux:

git credential-osxkeychain erase
host=github.com
protocol=https
[press Enter twice]

On Windows (use Git Bash):

git credential-manager clear
SOLUTION: SET NEW CREDENTIALS

Configure git with your current GitHub account email and username:

git config --global user.email "your-email@example.com"
git config --global user.name "Your GitHub Username"
VERIFY IT WORKED
git config --global user.email
git config --global user.name

Try push/pull again. If prompted, use your Personal Access Token (PAT) as password instead of your GitHub password.
